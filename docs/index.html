Vlodia ORM

Vlodia is a next-generation TypeScript ORM that combines the best features of modern ORMs with enterprise-grade performance and developer experience. Built for TypeScript-first development with reflection-driven metadata, database-agnostic architecture, and production-ready features.

Key Features

Reflection-Driven Architecture

  • Decorator-based metadata using @Entity, @Column, @Relation
  • Runtime type safety with TypeScript generics
  • Automatic schema inference from entity definitions
  • Zero configuration setup with sensible defaults

Database Agnostic

  • PostgreSQL with advanced features support
  • MySQL with full compatibility
  • SQLite for development and testing
  • Unified query interface across all databases
  • Database-specific optimizations when needed

Advanced Query Builder

  • Fluent API with method chaining
  • Type-safe queries with IntelliSense support
  • Complex joins and subqueries
  • Raw SQL support when needed
  • Query optimization and performance monitoring

Enterprise Features

  • Multi-level caching (L1 Identity Map + L2 Redis)
  • Transaction management with savepoints
  • Migration system with version control
  • Schema synchronization and validation
  • Comprehensive logging and diagnostics

Quick Start

Basic Entity Definition

import { Entity, Column, PrimaryKey, OneToMany, ManyToOne } from 'vlodia';

@Entity({ tableName: 'users' })
export class User {
  @PrimaryKey()
  id!: number;

  @Column({ unique: true })
  email!: string;

  @Column()
  name!: string;

  @Column({ nullable: true })
  age?: number;

  @OneToMany(() => Post, 'authorId')
  posts!: Post[];

  @Column({ default: () => new Date() })
  createdAt!: Date;
}

@Entity({ tableName: 'posts' })
export class Post {
  @PrimaryKey()
  id!: number;

  @Column()
  title!: string;

  @Column({ type: 'text' })
  content!: string;

  @Column()
  authorId!: number;

  @ManyToOne(() => User, 'authorId')
  author!: User;
}

ORM Configuration

import { Vlodia } from 'vlodia';

const orm = new Vlodia({
  database: {
    type: 'postgres',
    host: 'localhost',
    port: 5432,
    database: 'myapp',
    username: 'user',
    password: 'password',
    ssl: false
  },
  cache: {
    enabled: true,
    type: 'redis',
    ttl: 3600,
    maxSize: 10000
  },
  logging: {
    level: 'info',
    queries: true,
    slowQueries: 1000
  }
});

await orm.connect();

Basic Operations

// Create
const user = new User();
user.name = 'John Doe';
user.email = 'john@example.com';
await orm.manager.save(user);

// Read
const users = await orm.manager.find(User, {
  where: { age: { $gte: 18 } },
  orderBy: { name: 'ASC' },
  limit: 10
});

// Update
user.name = 'Jane Doe';
await orm.manager.save(user);

// Delete
await orm.manager.remove(user);

Advanced Queries

// Complex queries with joins
const posts = await orm.manager.find(Post, {
  where: {
    $and: [
      { title: { $like: '%TypeScript%' } },
      { author: { name: { $like: 'John%' } } }
    ]
  },
  relations: ['author'],
  orderBy: { createdAt: 'DESC' },
  limit: 20
});

// Raw SQL when needed
const results = await orm.manager.query(`
  SELECT u.name, COUNT(p.id) as post_count
  FROM users u
  LEFT JOIN posts p ON u.id = p.author_id
  GROUP BY u.id, u.name
  HAVING COUNT(p.id) > 5
`);

Transactions

await orm.transaction(async (manager) => {
  const user = new User();
  user.name = 'Transaction User';
  await manager.save(user);

  const post = new Post();
  post.title = 'My Post';
  post.authorId = user.id;
  await manager.save(post);

  // Automatic rollback on error
});

CLI Tools

Vlodia includes a powerful CLI for development:

# Generate entity from existing table
vlodia generate:entity --table users --output src/entities/User.ts

# Create new entity
vlodia create:entity --name Product --table products

# Generate migration
vlodia generate:migration --name add_user_roles

# Run migrations
vlodia migrate:run

# Schema synchronization
vlodia schema:sync --force

# Watch for changes
vlodia watch

Type Safety

Full TypeScript Support

// Type-safe entity operations
const user: User = await orm.manager.findOne(User, { where: { id: 1 } });

// Type-safe query building
const query = orm.manager
  .createQueryBuilder(User, 'u')
  .where('u.age > :age', { age: 18 })
  .orderBy('u.name', 'ASC')
  .getMany(); // Returns Promise<User[]>

// Type-safe relations
const userWithPosts = await orm.manager.findOne(User, {
  where: { id: 1 },
  relations: ['posts'] // TypeScript validates relation names
});

Migration System

Automatic Migrations

// Generate migration from entity changes
vlodia generate:migration --name add_user_roles

// Migration file
export class AddUserRoles1234567890 implements Migration {
  async up(queryRunner: QueryRunner): Promise<void> {
    await queryRunner.query(`
      ALTER TABLE users 
      ADD COLUMN role VARCHAR(50) DEFAULT 'user'
    `);
  }

  async down(queryRunner: QueryRunner): Promise<void> {
    await queryRunner.query(`
      ALTER TABLE users 
      DROP COLUMN role
    `);
  }
}

Caching System

// Multi-level caching
const user = await orm.manager.findOne(User, { 
  where: { id: 1 },
  cache: {
    ttl: 3600,
    tags: ['user', 'profile']

// Cache invalidation
await orm.cache.invalidateByTags(['user']);

Event System

// Entity lifecycle hooks
@Entity()
export class User {
  @BeforeInsert()
  beforeInsert() {
    this.createdAt = new Date();
  }

  @AfterUpdate()
  afterUpdate() {
    this.updatedAt = new Date();
  }
}

// Global events
orm.eventBus.on('query', (event) => {
  console.log('Query executed:', event.sql);
});

Validation

import { IsEmail, IsNotEmpty, MinLength } from 'class-validator';

@Entity()
export class User {
  @Column()
  @IsEmail()
  email!: string;

  @Column()
  @IsNotEmpty()
  @MinLength(2)
  name!: string;
}
